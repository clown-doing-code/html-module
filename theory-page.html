Introducción a HTML Semántico

HTML semántico es una forma de escribir código HTML utilizando etiquetas que describen claramente el propósito y el significado del contenido. Esto permite que navegadores, motores de búsqueda y tecnologías asistivas comprendan mejor la estructura de una página web.

¿Qué es la semántica en HTML?

La semántica se refiere al significado de las etiquetas que usamos en HTML. En lugar de utilizar etiquetas genéricas como div o span para todo, HTML semántico utiliza etiquetas específicas que describen el tipo de contenido que contienen, como header, footer, article, entre otras.

Ventajas del HTML Semántico

  1. Accesibilidad: Mejora la experiencia de los usuarios que utilizan lectores de pantalla u otras herramientas asistivas.
  2. SEO Optimizado: Los motores de búsqueda pueden entender mejor la estructura y el propósito del contenido.
  3. Código más legible: Es más fácil de mantener y comprender por otros desarrolladores.
  4. Compatibilidad con estándares web: Sigue las mejores prácticas definidas por el W3C.

Diferencia entre etiquetas semánticas y no semánticas

  • Etiquetas no semánticas:

    Son etiquetas genéricas que no aportan información sobre el contenido, como div y span.

  • Etiquetas semánticas:

    Son etiquetas que describen el propósito del contenido, como header, article y footer.

Ejemplos de etiquetas semánticas

A continuación, se describen algunas etiquetas semánticas comunes:

  • header: Se utiliza para la cabecera de una página o sección. Suele contener logotipos, títulos y menús de navegación.
  • nav: Define un bloque de enlaces de navegación.
  • main: Contiene el contenido principal de la página.
  • section: Agrupa contenido relacionado en bloques temáticos.
  • article Representa contenido independiente como un artículo de blog o una noticia.
  • aside: Contiene contenido relacionado con el contenido principal, como barras laterales o información adicional.
  • footer: Representa el pie de página de un documento o sección.
